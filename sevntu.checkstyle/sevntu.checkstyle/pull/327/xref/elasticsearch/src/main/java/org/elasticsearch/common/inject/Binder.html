29  /**
30   * Collects configuration information (primarily <i>bindings</i>) which will be
31   * used to create an {@link Injector}. Guice provides this object to your
32   * application's {@link Module} implementors so they may each contribute
33   * their own bindings and other registrations.
34   * <p/>
35   * <h3>The Guice Binding EDSL</h3>
36   * <p/>
37   * Guice uses an <i>embedded domain-specific language</i>, or EDSL, to help you
38   * create bindings simply and readably.  This approach is great for overall
39   * usability, but it does come with a small cost: <b>it is difficult to
40   * learn how to use the Binding EDSL by reading
41   * method-level javadocs</b>.  Instead, you should consult the series of
42   * examples below.  To save space, these examples omit the opening
43   * {@code binder}, just as you will if your module extends
44   * {@link AbstractModule}.
45   * <p/>
46   * <pre>
47   *     bind(ServiceImpl.class);</pre>
48   *
49   * This statement does essentially nothing; it "binds the {@code ServiceImpl}
50   * class to itself" and does not change Guice's default behavior.  You may still
51   * want to use this if you prefer your {@link Module} class to serve as an
52   * explicit <i>manifest</i> for the services it provides.  Also, in rare cases,
53   * Guice may be unable to validate a binding at injector creation time unless it
54   * is given explicitly.
55   *
56   * <pre>
57   *     bind(Service.class).to(ServiceImpl.class);</pre>
58   *
59   * Specifies that a request for a {@code Service} instance with no binding
60   * annotations should be treated as if it were a request for a
61   * {@code ServiceImpl} instance. This <i>overrides</i> the function of any
62   * {@link ImplementedBy @ImplementedBy} or {@link ProvidedBy @ProvidedBy}
63   * annotations found on {@code Service}, since Guice will have already
64   * "moved on" to {@code ServiceImpl} before it reaches the point when it starts
65   * looking for these annotations.
66   *
67   * <pre>
68   *     bind(Service.class).toProvider(ServiceProvider.class);</pre>
69   *
70   * In this example, {@code ServiceProvider} must extend or implement
71   * {@code Provider<Service>}. This binding specifies that Guice should resolve
72   * an unannotated injection request for {@code Service} by first resolving an
73   * instance of {@code ServiceProvider} in the regular way, then calling
74   * {@link Provider#get get()} on the resulting Provider instance to obtain the
75   * {@code Service} instance.
76   *
77   * <p>The {@link Provider} you use here does not have to be a "factory"; that
78   * is, a provider which always <i>creates</i> each instance it provides.
79   * However, this is generally a good practice to follow.  You can then use
80   * Guice's concept of {@link Scope scopes} to guide when creation should happen
81   * -- "letting Guice work for you".
82   *
83   * <pre>
84   *     bind(Service.class).annotatedWith(Red.class).to(ServiceImpl.class);</pre>
85   *
86   * Like the previous example, but only applies to injection requests that use
87   * the binding annotation {@code @Red}.  If your module also includes bindings
88   * for particular <i>values</i> of the {@code @Red} annotation (see below),
89   * then this binding will serve as a "catch-all" for any values of {@code @Red}
90   * that have no exact match in the bindings.
91   *
92   * <pre>
93   *     bind(ServiceImpl.class).in(Singleton.class);
94   *     // or, alternatively
95   *     bind(ServiceImpl.class).in(Scopes.SINGLETON);</pre>
96   *
97   * Either of these statements places the {@code ServiceImpl} class into
98   * singleton scope.  Guice will create only one instance of {@code ServiceImpl}
99   * and will reuse it for all injection requests of this type.  Note that it is
100  * still possible to bind another instance of {@code ServiceImpl} if the second
101  * binding is qualified by an annotation as in the previous example.  Guice is
102  * not overly concerned with <i>preventing</i> you from creating multiple
103  * instances of your "singletons", only with <i>enabling</i> your application to
104  * share only one instance if that's all you tell Guice you need.
105  *
106  * <p><b>Note:</b> a scope specified in this way <i>overrides</i> any scope that
107  * was specified with an annotation on the {@code ServiceImpl} class.
108  *
109  * <p>Besides {@link Singleton}/{@link Scopes#SINGLETON}, there are
110  * servlet-specific scopes available in
111  * {@code com.google.inject.servlet.ServletScopes}, and your Modules can
112  * contribute their own custom scopes for use here as well.
113  *
114  * <pre>
115  *     bind(new TypeLiteral&lt;PaymentService&lt;CreditCard>>() {})
116  *         .to(CreditCardPaymentService.class);</pre>
117  *
118  * This admittedly odd construct is the way to bind a parameterized type. It
119  * tells Guice how to honor an injection request for an element of type
120  * {@code PaymentService<CreditCard>}. The class
121  * {@code CreditCardPaymentService} must implement the
122  * {@code PaymentService<CreditCard>} interface.  Guice cannot currently bind or
123  * inject a generic type, such as {@code Set<E>}; all type parameters must be
124  * fully specified.
125  *
126  * <pre>
127  *     bind(Service.class).toInstance(new ServiceImpl());
128  *     // or, alternatively
129  *     bind(Service.class).toInstance(SomeLegacyRegistry.getService());</pre>
130  *
131  * In this example, your module itself, <i>not Guice</i>, takes responsibility
132  * for obtaining a {@code ServiceImpl} instance, then asks Guice to always use
133  * this single instance to fulfill all {@code Service} injection requests.  When
134  * the {@link Injector} is created, it will automatically perform field
135  * and method injection for this instance, but any injectable constructor on
136  * {@code ServiceImpl} is simply ignored.  Note that using this approach results
137  * in "eager loading" behavior that you can't control.
138  *
139  * <pre>
140  *     bindConstant().annotatedWith(ServerHost.class).to(args[0]);</pre>
141  *
142  * Sets up a constant binding. Constant injections must always be annotated.
143  * When a constant binding's value is a string, it is eligile for conversion to
144  * all primitive types, to {@link Enum#valueOf(Class, String) all enums}, and to
145  * {@link Class#forName class literals}. Conversions for other types can be
146  * configured using {@link #convertToTypes(Matcher, TypeConverter)
147  * convertToTypes()}.
148  *
149  * <pre>
150  *   {@literal @}Color("red") Color red; // A member variable (field)
151  *    . . .
152  *     red = MyModule.class.getDeclaredField("red").getAnnotation(Color.class);
153  *     bind(Service.class).annotatedWith(red).to(RedService.class);</pre>
154  *
155  * If your binding annotation has parameters you can apply different bindings to
156  * different specific values of your annotation.  Getting your hands on the
157  * right instance of the annotation is a bit of a pain -- one approach, shown
158  * above, is to apply a prototype annotation to a field in your module class, so
159  * that you can read this annotation instance and give it to Guice.
160  *
161  * <pre>
162  *     bind(Service.class)
163  *         .annotatedWith(Names.named("blue"))
164  *         .to(BlueService.class);</pre>
165  *
166  * Differentiating by names is a common enough use case that we provided a
167  * standard annotation, {@link org.elasticsearch.common.inject.name.Named @Named}.  Because of
168  * Guice's library support, binding by name is quite easier than in the
169  * arbitrary binding annotation case we just saw.  However, remember that these
170  * names will live in a single flat namespace with all the other names used in
171  * your application.
172  *
173  * <p>The above list of examples is far from exhaustive.  If you can think of
174  * how the concepts of one example might coexist with the concepts from another,
175  * you can most likely weave the two together.  If the two concepts make no
176  * sense with each other, you most likely won't be able to do it.  In a few
177  * cases Guice will let something bogus slip by, and will then inform you of
178  * the problems at runtime, as soon as you try to create your Injector.
179  *
180  * <p>The other methods of Binder such as {@link #bindScope},
181  * {@link #bindInterceptor}, {@link #install}, {@link #requestStaticInjection},
182  * {@link #addError} and {@link #currentStage} are not part of the Binding EDSL;
183  * you can learn how to use these in the usual way, from the method
184  * documentation.
185  *
186  * @author crazybob@google.com (Bob Lee)
187  * @author jessewilson@google.com (Jesse Wilson)
188  * @author kevinb@google.com (Kevin Bourrillion)
189  */
190 public interface Binder {
191 
192     /**
193      * Binds a scope to an annotation.
194      */
195     void bindScope(Class<? extends Annotation> annotationType, Scope scope);
196 
197     /**
198      * See the EDSL examples at {@link Binder}.
199      */
200     <T> LinkedBindingBuilder<T> bind(Key<T> key);
201 
202     /**
203      * See the EDSL examples at {@link Binder}.
204      */
205     <T> AnnotatedBindingBuilder<T> bind(TypeLiteral<T> typeLiteral);
206 
207     /**
208      * See the EDSL examples at {@link Binder}.
209      */
210     <T> AnnotatedBindingBuilder<T> bind(Class<T> type);
211 
212     /**
213      * See the EDSL examples at {@link Binder}.
214      */
215     AnnotatedConstantBindingBuilder bindConstant();
216 
217     /**
218      * Upon successful creation, the {@link Injector} will inject instance fields
219      * and methods of the given object.
220      *
221      * @param type     of instance
222      * @param instance for which members will be injected
223      * @since 2.0
224      */
225     <T> void requestInjection(TypeLiteral<T> type, T instance);
226 
227     /**
228      * Upon successful creation, the {@link Injector} will inject instance fields
229      * and methods of the given object.
230      *
231      * @param instance for which members will be injected
232      * @since 2.0
233      */
234     void requestInjection(Object instance);
235 
236     /**
237      * Upon successful creation, the {@link Injector} will inject static fields
238      * and methods in the given classes.
239      *
240      * @param types for which static members will be injected
241      */
242     void requestStaticInjection(Class<?>... types);
243 
244     /**
245      * Uses the given module to configure more bindings.
246      */
247     void install(Module module);
248 
249     /**
250      * Gets the current stage.
251      */
252     Stage currentStage();
253 
254     /**
255      * Records an error message which will be presented to the user at a later
256      * time. Unlike throwing an exception, this enable us to continue
257      * configuring the Injector and discover more errors. Uses {@link
258      * String#format(String, Object[])} to insert the arguments into the
259      * message.
260      */
261     void addError(String message, Object... arguments);
262 
263     /**
264      * Records an exception, the full details of which will be logged, and the
265      * message of which will be presented to the user at a later
266      * time. If your Module calls something that you worry may fail, you should
267      * catch the exception and pass it into this.
268      */
269     void addError(Throwable t);
270 
271     /**
272      * Records an error message to be presented to the user at a later time.
273      *
274      * @since 2.0
275      */
276     void addError(Message message);
277 
278     /**
279      * Returns the provider used to obtain instances for the given injection key.
280      * The returned will not be valid until the {@link Injector} has been
281      * created. The provider will throw an {@code IllegalStateException} if you
282      * try to use it beforehand.
283      *
284      * @since 2.0
285      */
286     <T> Provider<T> getProvider(Key<T> key);
287 
288     /**
289      * Returns the provider used to obtain instances for the given injection type.
290      * The returned provider will not be valid until the {@link Injector} has been
291      * created. The provider will throw an {@code IllegalStateException} if you
292      * try to use it beforehand.
293      *
294      * @since 2.0
295      */
296     <T> Provider<T> getProvider(Class<T> type);
297 
298     /**
299      * Returns the members injector used to inject dependencies into methods and fields on instances
300      * of the given type {@code T}. The returned members injector will not be valid until the main
301      * {@link Injector} has been created. The members injector will throw an {@code
302      * IllegalStateException} if you try to use it beforehand.
303      *
304      * @param typeLiteral type to get members injector for
305      * @since 2.0
306      */
307     <T> MembersInjector<T> getMembersInjector(TypeLiteral<T> typeLiteral);
308 
309     /**
310      * Returns the members injector used to inject dependencies into methods and fields on instances
311      * of the given type {@code T}. The returned members injector will not be valid until the main
312      * {@link Injector} has been created. The members injector will throw an {@code
313      * IllegalStateException} if you try to use it beforehand.
314      *
315      * @param type type to get members injector for
316      * @since 2.0
317      */
318     <T> MembersInjector<T> getMembersInjector(Class<T> type);
319 
320     /**
321      * Binds a type converter. The injector will use the given converter to
322      * convert string constants to matching types as needed.
323      *
324      * @param typeMatcher matches types the converter can handle
325      * @param converter   converts values
326      * @since 2.0
327      */
328     void convertToTypes(Matcher<? super TypeLiteral<?>> typeMatcher,
329                         TypeConverter converter);
330 
331     /**
332      * Registers a listener for injectable types. Guice will notify the listener when it encounters
333      * injectable types matched by the given type matcher.
334      *
335      * @param typeMatcher that matches injectable types the listener should be notified of
336      * @param listener    for injectable types matched by typeMatcher
337      * @since 2.0
338      */
339     void bindListener(Matcher<? super TypeLiteral<?>> typeMatcher,
340                       TypeListener listener);
341 
342     /**
343      * Returns a binder that uses {@code source} as the reference location for
344      * configuration errors. This is typically a {@link StackTraceElement}
345      * for {@code .java} source but it could any binding source, such as the
346      * path to a {@code .properties} file.
347      *
348      * @param source any object representing the source location and has a
349      *               concise {@link Object#toString() toString()} value
350      * @return a binder that shares its configuration with this binder
351      * @since 2.0
352      */
353     Binder withSource(Object source);
354 
355     /**
356      * Returns a binder that skips {@code classesToSkip} when identify the
357      * calling code. The caller's {@link StackTraceElement} is used to locate
358      * the source of configuration errors.
359      *
360      * @param classesToSkip library classes that create bindings on behalf of
361      *                      their clients.
362      * @return a binder that shares its configuration with this binder.
363      * @since 2.0
364      */
365     Binder skipSources(Class... classesToSkip);
366 
367     /**
368      * Creates a new private child environment for bindings and other configuration. The returned
369      * binder can be used to add and configuration information in this environment. See {@link
370      * PrivateModule} for details.
371      *
372      * @return a binder that inherits configuration from this binder. Only exposed configuration on
373      *         the returned binder will be visible to this binder.
374      * @since 2.0
375      */
376     PrivateBinder newPrivateBinder();
377 }
