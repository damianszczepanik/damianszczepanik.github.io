39  /**
40   * Abstract base class for message listener containers. Can either host
41   * a standard JMS {@link javax.jms.MessageListener} or a Spring-specific
42   * {@link SessionAwareMessageListener}.
43   *
44   * <p>Usually holds a single JMS {@link Connection} that all listeners are
45   * supposed to be registered on, which is the standard JMS way of managing
46   * listeners. Can alternatively also be used with a fresh Connection per
47   * listener, for J2EE-style XA-aware JMS messaging. The actual registration
48   * process is up to concrete subclasses.
49   *
50   * <p><b>NOTE:</b> The default behavior of this message listener container
51   * is to <b>never</b> propagate an exception thrown by a message listener up to
52   * the JMS provider. Instead, it will log any such exception at the error level.
53   * This means that from the perspective of the attendant JMS provider no such
54   * listener will ever fail. However, if error handling is necessary, then
55   * any implementation of the {@link ErrorHandler} strategy may be provided to
56   * the {@link #setErrorHandler(ErrorHandler)} method. Note that JMSExceptions
57   * <b>will</b> be passed to the ErrorHandler in addition to (but after) being
58   * passed to an {@link ExceptionListener}, if one has been provided.
59   *
60   * <p>The listener container offers the following message acknowledgment options:
61   * <ul>
62   * <li>"sessionAcknowledgeMode" set to "AUTO_ACKNOWLEDGE" (default):
63   * This mode is container-dependent: For {@link DefaultMessageListenerContainer},
64   * it means automatic message acknowledgment <i>before</i> listener execution, with
65   * no redelivery in case of an exception. For {@link SimpleMessageListenerContainer},
66   * it means automatic message acknowledgment <i>after</i> listener execution, with
67   * redelivery in case of an exception thrown, as defined by the JMS specification.
68   * In order to consistently achieve the latter behavior with any container variant,
69   * consider setting "sessionTransacted" to "true" instead.
70   * <li>"sessionAcknowledgeMode" set to "CLIENT_ACKNOWLEDGE":
71   * Automatic message acknowledgment <i>after</i> successful listener execution;
72   * no redelivery in case of exception thrown.
73   * <li>"sessionAcknowledgeMode" set to "DUPS_OK_ACKNOWLEDGE":
74   * <i>Lazy</i> message acknowledgment during or after listener execution;
75   * <i>potential redelivery</i> in case of exception thrown.
76   * <li>"sessionTransacted" set to "true":
77   * Transactional acknowledgment after successful listener execution;
78   * <i>guaranteed redelivery</i> in case of exception thrown.
79   * </ul>
80   * The exact behavior might vary according to the concrete listener container
81   * and JMS provider used.
82   *
83   * <p>There are two solutions to the duplicate processing problem:
84   * <ul>
85   * <li>Either add <i>duplicate message detection</i> to your listener, in the
86   * form of a business entity existence check or a protocol table check. This
87   * usually just needs to be done in case of the JMSRedelivered flag being
88   * set on the incoming message (else just process straightforwardly).
89   * <li>Or wrap the <i>entire processing with an XA transaction</i>, covering the
90   * reception of the message as well as the execution of the message listener.
91   * This is only supported by {@link DefaultMessageListenerContainer}, through
92   * specifying a "transactionManager" (typically a
93   * {@link org.springframework.transaction.jta.JtaTransactionManager}, with
94   * a corresponding XA-aware JMS {@link javax.jms.ConnectionFactory} passed in as
95   * "connectionFactory").
96   * </ul>
97   * Note that XA transaction coordination adds significant runtime overhead,
98   * so it might be feasible to avoid it unless absolutely necessary.
99   *
100  * <p><b>Recommendations:</b>
101  * <ul>
102  * <li>The general recommendation is to set "sessionTransacted" to "true",
103  * typically in combination with local database transactions triggered by the
104  * listener implementation, through Spring's standard transaction facilities.
105  * This will work nicely in Tomcat or in a standalone environment, often
106  * combined with custom duplicate message detection (if it is unacceptable
107  * to ever process the same message twice).
108  * <li>Alternatively, specify a
109  * {@link org.springframework.transaction.jta.JtaTransactionManager} as
110  * "transactionManager" for a fully XA-aware JMS provider - typically when
111  * running on a J2EE server, but also for other environments with a JTA
112  * transaction manager present. This will give full "exactly-once" guarantees
113  * without custom duplicate message checks, at the price of additional
114  * runtime processing overhead.
115  * </ul>
116  *
117  * <p>Note that the "sessionTransacted" flag is strongly recommended over
118  * {@link org.springframework.jms.connection.JmsTransactionManager}, provided
119  * that transactions do not need to be managed externally. As a consequence,
120  * set the transaction manager only if you are using JTA or if you need to
121  * synchronize with custom external transaction arrangements.
122  *
123  * @author Juergen Hoeller
124  * @author Stephane Nicoll
125  * @since 2.0
126  * @see #setMessageListener
127  * @see javax.jms.MessageListener
128  * @see SessionAwareMessageListener
129  * @see #handleListenerException
130  * @see DefaultMessageListenerContainer
131  * @see SimpleMessageListenerContainer
132  * @see org.springframework.jms.listener.endpoint.JmsMessageEndpointManager
133  */

135 		extends AbstractJmsListeningContainer implements MessageListenerContainer {

554 	/**
555 	 * Execute the specified listener,
556 	 * committing or rolling back the transaction afterwards (if necessary).
557 	 * @param session the JMS Session to operate on
558 	 * @param message the received JMS Message
559 	 * @see #invokeListener
560 	 * @see #commitIfNecessary
561 	 * @see #rollbackOnExceptionIfNecessary
562 	 * @see #handleListenerException
563 	 */
564 	protected void executeListener(Session session, Message message) {
565 		try {
566 			doExecuteListener(session, message);
567 		}
568 		catch (Throwable ex) {
569 			handleListenerException(ex);
570 		}
571 	}
572 
573 	/**
574 	 * Execute the specified listener,
575 	 * committing or rolling back the transaction afterwards (if necessary).
576 	 * @param session the JMS Session to operate on
577 	 * @param message the received JMS Message
578 	 * @throws JMSException if thrown by JMS API methods
579 	 * @see #invokeListener
580 	 * @see #commitIfNecessary
581 	 * @see #rollbackOnExceptionIfNecessary
582 	 * @see #convertJmsAccessException
583 	 */
584 	protected void doExecuteListener(Session session, Message message) throws JMSException {
585 		if (!isAcceptMessagesWhileStopping() && !isRunning()) {
586 			if (logger.isWarnEnabled()) {
587 				logger.warn("Rejecting received message because of the listener container " +
588 						"having been stopped in the meantime: " + message);
589 			}
590 			rollbackIfNecessary(session);
591 			throw new MessageRejectedWhileStoppingException();
592 		}
593 		try {
594 			invokeListener(session, message);
595 		}
596 		catch (JMSException ex) {
597 			rollbackOnExceptionIfNecessary(session, ex);
598 			throw ex;
599 		}
600 		catch (RuntimeException ex) {
601 			rollbackOnExceptionIfNecessary(session, ex);
602 			throw ex;
603 		}
604 		catch (Error err) {
605 			rollbackOnExceptionIfNecessary(session, err);
606 			throw err;
607 		}
608 		commitIfNecessary(session, message);
609 	}

611 	/**
612 	 * Invoke the specified listener: either as standard JMS MessageListener
613 	 * or (preferably) as Spring SessionAwareMessageListener.
614 	 * @param session the JMS Session to operate on
615 	 * @param message the received JMS Message
616 	 * @throws JMSException if thrown by JMS API methods
617 	 * @see #setMessageListener
618 	 */
619 	@SuppressWarnings("rawtypes")
620 	protected void invokeListener(Session session, Message message) throws JMSException {
621 		Object listener = getMessageListener();
622 		if (listener instanceof SessionAwareMessageListener) {
623 			doInvokeListener((SessionAwareMessageListener) listener, session, message);
624 		}
625 		else if (listener instanceof MessageListener) {
626 			doInvokeListener((MessageListener) listener, message);
627 		}
628 		else if (listener != null) {
629 			throw new IllegalArgumentException(
630 					"Only MessageListener and SessionAwareMessageListener supported: " + listener);
631 		}
632 		else {
633 			throw new IllegalStateException("No message listener specified - see property 'messageListener'");
634 		}
635 	}

638 	 * Invoke the specified listener as Spring SessionAwareMessageListener,
639 	 * exposing a new JMS Session (potentially with its own transaction)
640 	 * to the listener if demanded.
641 	 * @param listener the Spring SessionAwareMessageListener to invoke
642 	 * @param session the JMS Session to operate on
643 	 * @param message the received JMS Message
644 	 * @throws JMSException if thrown by JMS API methods
645 	 * @see SessionAwareMessageListener
646 	 * @see #setExposeListenerSession
647 	 */
648 	@SuppressWarnings({ "unchecked", "rawtypes" })
649 	protected void doInvokeListener(SessionAwareMessageListener listener, Session session, Message message)
650 			throws JMSException {
651 
652 		Connection conToClose = null;
653 		Session sessionToClose = null;
654 		try {
655 			Session sessionToUse = session;
656 			if (!isExposeListenerSession()) {
657 				// We need to expose a separate Session.
658 				conToClose = createConnection();
659 				sessionToClose = createSession(conToClose);
660 				sessionToUse = sessionToClose;
661 			}
662 			// Actually invoke the message listener...
663 			listener.onMessage(message, sessionToUse);
664 			// Clean up specially exposed Session, if any.
665 			if (sessionToUse != session) {
666 				if (sessionToUse.getTransacted() && isSessionLocallyTransacted(sessionToUse)) {
667 					// Transacted session created by this container -> commit.
668 					JmsUtils.commitIfNecessary(sessionToUse);
669 				}
670 			}
671 		}
672 		finally {
673 			JmsUtils.closeSession(sessionToClose);
674 			JmsUtils.closeConnection(conToClose);
675 		}
676 	}

692 	 * Perform a commit or message acknowledgement, as appropriate.
693 	 * @param session the JMS Session to commit
694 	 * @param message the Message to acknowledge
695 	 * @throws javax.jms.JMSException in case of commit failure
696 	 */
697 	protected void commitIfNecessary(Session session, Message message) throws JMSException {
698 		// Commit session or acknowledge message.
699 		if (session.getTransacted()) {
700 			// Commit necessary - but avoid commit call within a JTA transaction.
701 			if (isSessionLocallyTransacted(session)) {
702 				// Transacted session created by this container -> commit.
703 				JmsUtils.commitIfNecessary(session);
704 			}
705 		}
706 		else if (message != null && isClientAcknowledge(session)) {
707 			message.acknowledge();
708 		}
709 	}

712 	 * Perform a rollback, if appropriate.
713 	 * @param session the JMS Session to rollback
714 	 * @throws javax.jms.JMSException in case of a rollback error
715 	 */
716 	protected void rollbackIfNecessary(Session session) throws JMSException {
717 		if (session.getTransacted()) {
718 			if (isSessionLocallyTransacted(session)) {
719 				// Transacted session created by this container -> rollback.
720 				JmsUtils.rollbackIfNecessary(session);
721 			}
722 		}
723 		else {
724 			session.recover();
725 		}
726 	}

728 	/**
729 	 * Perform a rollback, handling rollback exceptions properly.
730 	 * @param session the JMS Session to rollback
731 	 * @param ex the thrown application exception or error
732 	 * @throws javax.jms.JMSException in case of a rollback error
733 	 */
734 	protected void rollbackOnExceptionIfNecessary(Session session, Throwable ex) throws JMSException {
735 		try {
736 			if (session.getTransacted()) {
737 				if (isSessionLocallyTransacted(session)) {
738 					// Transacted session created by this container -> rollback.
739 					if (logger.isDebugEnabled()) {
740 						logger.debug("Initiating transaction rollback on application exception", ex);
741 					}
742 					JmsUtils.rollbackIfNecessary(session);
743 				}
744 			}
745 			else {
746 				session.recover();
747 			}
748 		}
749 		catch (IllegalStateException ex2) {
750 			logger.debug("Could not roll back because Session already closed", ex2);
751 		}
752 		catch (JMSException ex2) {
753 			logger.error("Application exception overridden by rollback exception", ex);
754 			throw ex2;
755 		}
756 		catch (RuntimeException ex2) {
757 			logger.error("Application exception overridden by rollback exception", ex);
758 			throw ex2;
759 		}
760 		catch (Error err) {
761 			logger.error("Application exception overridden by rollback error", ex);
762 			throw err;
763 		}
764 	}

782 	/**
783 	 * Create a JMS MessageConsumer for the given Session and Destination.
784 	 * <p>This implementation uses JMS 1.1 API.
785 	 * @param session the JMS Session to create a MessageConsumer for
786 	 * @param destination the JMS Destination to create a MessageConsumer for
787 	 * @return the new JMS MessageConsumer
788 	 * @throws javax.jms.JMSException if thrown by JMS API methods
789 	 */
790 	protected MessageConsumer createConsumer(Session session, Destination destination) throws JMSException {
791 		if (isPubSubDomain() && destination instanceof Topic) {
792 			if (isSubscriptionShared()) {
793 				// createSharedConsumer((Topic) dest, subscription, selector);
794 				// createSharedDurableConsumer((Topic) dest, subscription, selector);
795 				Method method = (isSubscriptionDurable() ?
796 						createSharedDurableConsumerMethod : createSharedConsumerMethod);
797 				try {
798 					return (MessageConsumer) method.invoke(session, destination, getSubscriptionName(), getMessageSelector());
799 				}
800 				catch (InvocationTargetException ex) {
801 					if (ex.getTargetException() instanceof JMSException) {
802 						throw (JMSException) ex.getTargetException();
803 					}
804 					ReflectionUtils.handleInvocationTargetException(ex);
805 					return null;
806 				}
807 				catch (IllegalAccessException ex) {
808 					throw new IllegalStateException("Could not access JMS 2.0 API method: " + ex.getMessage());
809 				}
810 			}
811 			else if (isSubscriptionDurable()) {
812 				return session.createDurableSubscriber(
813 						(Topic) destination, getSubscriptionName(), getMessageSelector(), isPubSubNoLocal());
814 			}
815 			else {
816 				// Only pass in the NoLocal flag in case of a Topic (pub-sub mode):
817 				// Some JMS providers, such as WebSphere MQ 6.0, throw IllegalStateException
818 				// in case of the NoLocal flag being specified for a Queue.
819 				return session.createConsumer(destination, getMessageSelector(), isPubSubNoLocal());
820 			}
821 		}
822 		else {
823 			return session.createConsumer(destination, getMessageSelector());
824 		}
825 	}

827 	/**
828 	 * Handle the given exception that arose during listener execution.
829 	 * <p>The default implementation logs the exception at warn level,
830 	 * not propagating it to the JMS provider &mdash; assuming that all handling of
831 	 * acknowledgement and/or transactions is done by this listener container.
832 	 * This can be overridden in subclasses.
833 	 * @param ex the exception to handle
834 	 */
835 	protected void handleListenerException(Throwable ex) {
836 		if (ex instanceof MessageRejectedWhileStoppingException) {
837 			// Internal exception - has been handled before.
838 			return;
839 		}
840 		if (ex instanceof JMSException) {
841 			invokeExceptionListener((JMSException) ex);
842 		}
843 		if (isActive()) {
844 			// Regular case: failed while active.
845 			// Invoke ErrorHandler if available.
846 			invokeErrorHandler(ex);
847 		}
848 		else {
849 			// Rare case: listener thread failed after container shutdown.
850 			// Log at debug level, to avoid spamming the shutdown log.
851 			logger.debug("Listener exception after container shutdown", ex);
852 		}
853 	}
854 
855 	/**
856 	 * Invoke the registered JMS ExceptionListener, if any.
857 	 * @param ex the exception that arose during JMS processing
858 	 * @see #setExceptionListener
859 	 */
860 	protected void invokeExceptionListener(JMSException ex) {
861 		ExceptionListener exceptionListener = getExceptionListener();
862 		if (exceptionListener != null) {
863 			exceptionListener.onException(ex);
864 		}
865 	}
866 
867 	/**
868 	 * Invoke the registered ErrorHandler, if any. Log at warn level otherwise.
869 	 * @param ex the uncaught error that arose during JMS processing.
870 	 * @see #setErrorHandler
871 	 */
872 	protected void invokeErrorHandler(Throwable ex) {
873 		ErrorHandler errorHandler = getErrorHandler();
874 		if (errorHandler != null) {
875 			errorHandler.handleError(ex);
876 		}
877 		else if (logger.isWarnEnabled()) {
878 			logger.warn("Execution of JMS message listener failed, and no ErrorHandler has been set.", ex);
879 		}
880 	}
